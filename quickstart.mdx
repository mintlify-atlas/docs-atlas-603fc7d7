---
title: "Quickstart: clone and run Lebrijaleo on your machine"
description: "Learn how to clone the Lebrijaleo repository, install its dependencies, and start the Next.js development server in under five minutes."
---

This page walks you through cloning the Lebrijaleo repository, installing its dependencies, and starting the development server so you can run the project on your machine.

<Steps>
  <Step title="Clone the repository">
    Clone the TFG repository from GitHub:

    ```bash
    git clone https://github.com/alcorher/TFG
    ```
  </Step>

  <Step title="Navigate to the project directory">
    Move into the Lebrijaleo application folder:

    ```bash
    cd TFG/lebrijaleo
    ```
  </Step>

  <Step title="Install dependencies">
    Install the project's dependencies using your preferred package manager:

    <CodeGroup>

    ```bash npm
    npm install
    ```

    ```bash yarn
    yarn install
    ```

    ```bash pnpm
    pnpm install
    ```

    ```bash bun
    bun install
    ```

    </CodeGroup>
  </Step>

  <Step title="Start the development server">
    Run the dev script defined in `package.json` (`next dev`):

    <CodeGroup>

    ```bash npm
    npm run dev
    ```

    ```bash yarn
    yarn dev
    ```

    ```bash pnpm
    pnpm dev
    ```

    ```bash bun
    bun dev
    ```

    </CodeGroup>
  </Step>

  <Step title="Open the app in your browser">
    Navigate to [http://localhost:3000](http://localhost:3000). You should see the Lebrijaleo home page.

    <Note>
      The page auto-updates as you edit `src/app/page.js`. You do not need to restart the server after saving a change.
    </Note>
  </Step>
</Steps>

## Production build

Once you are ready to build for production, use the `build` and `start` scripts:

<CodeGroup>

```bash npm
npm run build
npm run start
```

```bash yarn
yarn build
yarn start
```

```bash pnpm
pnpm build
pnpm start
```

```bash bun
bun run build
bun run start
```

</CodeGroup>

`next build` compiles and optimises the application. `next start` then serves the compiled output. Run `build` before `start` every time you deploy a new version.
